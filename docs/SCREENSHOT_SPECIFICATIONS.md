# Kull App Store Screenshot Specifications

## Overview

High-quality screenshots are critical for App Store conversion. This guide covers required sizes, design principles, and content strategy for Kull's App Store listing.

## Required Screenshot Sizes

### iPhone Screenshots

**iPhone 6.7" Display (Pro Max models)** - REQUIRED
- **Resolution:** 1290 x 2796 pixels (portrait) or 2796 x 1290 pixels (landscape)
- **Devices:** iPhone 14 Pro Max, 15 Pro Max, 16 Pro Max
- **Quantity:** 3-10 screenshots

**iPhone 6.5" Display** - REQUIRED
- **Resolution:** 1242 x 2688 pixels (portrait) or 2688 x 1242 pixels (landscape)
- **Devices:** iPhone 11 Pro Max, XS Max
- **Quantity:** 3-10 screenshots

**iPhone 5.5" Display (Optional, but recommended for older device support)**
- **Resolution:** 1242 x 2208 pixels (portrait) or 2208 x 1242 pixels (landscape)
- **Devices:** iPhone 8 Plus, 7 Plus, 6s Plus
- **Quantity:** 3-10 screenshots

### iPad Screenshots

**iPad Pro 12.9" Display (3rd gen)** - REQUIRED
- **Resolution:** 2048 x 2732 pixels (portrait) or 2732 x 2048 pixels (landscape)
- **Devices:** iPad Pro 12.9" (3rd gen and later)
- **Quantity:** 3-10 screenshots

**iPad Pro 11" Display (Optional)**
- **Resolution:** 1668 x 2388 pixels (portrait) or 2388 x 1668 pixels (landscape)
- **Devices:** iPad Pro 11"
- **Quantity:** 3-10 screenshots

### macOS Screenshots

**macOS** - REQUIRED
- **Minimum Resolution:** 1280 x 800 pixels
- **Maximum Resolution:** 3840 x 2400 pixels
- **Recommended:** 2880 x 1800 pixels (Retina MacBook Pro 16" native)
- **Aspect Ratio:** 16:10 (standard macOS)
- **Quantity:** 1-10 screenshots

## Design Guidelines

### Visual Style

**Color Palette:**
- **Primary:** Deep blue (#0066CC) - Kull brand color
- **Accent:** Electric cyan (#00D4FF) - Highlights, CTAs
- **Neutral:** Light gray backgrounds (#F5F5F5), dark text (#1A1A1A)
- **Success:** Green (#00CC66) - Completed actions
- **Warning:** Yellow (#FFB800) - 3-star ratings
- **Error:** Red (#FF3B30) - Rejected photos

**Typography:**
- **Headline:** SF Pro Display Bold, 48-64pt
- **Subtext:** SF Pro Text Regular, 24-32pt
- **UI Elements:** SF Pro Text (system font for authenticity)

**Layout:**
- **White Space:** Generous margins (80-120px from edges)
- **Hierarchy:** Clear headline → subtext → screenshot → annotation
- **Annotations:** Subtle arrows or highlights (not overwhelming)

### Screenshot Frame

**Option 1: Device Frame (Recommended)**
- Use realistic iPhone/iPad/Mac frame
- Shows app in context (users understand platform)
- Tools: [Previewed](https://previewed.app/), [Screely](https://screely.com/)

**Option 2: Frameless with Background (Alternative)**
- Full-bleed screenshot with gradient background
- More space for headline/subtext overlays
- Tools: Figma, Sketch, Photoshop

**Option 3: Hybrid (Best for macOS)**
- Mac window frame with transparent background
- Add gradient or subtle pattern behind

### Overlay Text

**Headline (Large, Bold):**
- 1-5 words, sentence case or title case
- Communicate value, not feature
- Examples:
  - "1,000 Photos in 30 Seconds" (value)
  - "Lightroom-Ready in One Click" (value)
  - NOT "AI Processing Dashboard" (feature)

**Subtext (Smaller, Regular Weight):**
- 5-12 words, sentence case
- Explain how the value is achieved
- Examples:
  - "AI-powered culling that matches professional photographers"
  - "XMP sidecar files automatically recognized by Lightroom"

**Placement:**
- **Top:** Headline + subtext above screenshot (most common)
- **Bottom:** Headline + subtext below screenshot
- **Overlay:** Text directly on screenshot (ensure high contrast)

### Annotations

**When to Use:**
- Highlight specific UI elements (e.g., "Star ratings here")
- Show interaction flow (e.g., arrows pointing from drag-drop to results)
- Explain non-obvious features (e.g., "AI analyzes 12 quality dimensions")

**Best Practices:**
- **Subtle:** Don't overwhelm the screenshot
- **Contrast:** Use arrows/circles that stand out but don't clash
- **Consistency:** Same annotation style across all screenshots

## Screenshot Production Workflow

### Step 1: Prepare Sample Data

Create realistic demo photoshoot:
- 1,247 photos (common wedding shoot size)
- Mix of 5-star (hero shots), 4-star (good), 3-star (maybe), 2-star (technical issues), 1-star (reject)
- XMP files generated for all photos

### Step 2: Capture Screenshots

**macOS:**
1. Open Kull menu bar app
2. Press Cmd+Shift+4, then Spacebar to capture window
3. Click on Kull window to capture
4. Repeat for all key screens (dashboard, settings, export dialog)

**iOS:**
1. Install Kull on physical device or simulator
2. Press Volume Up + Power button (physical device)
3. Or use Simulator → File → New Screen Shot
4. Capture dashboard, settings, notifications

**Lightroom Integration:**
1. Open Lightroom with sample photos + XMP files
2. Capture Lightroom library showing star ratings
3. Capture side-by-side with Kull export dialog

### Step 3: Design in Figma/Sketch

**Template Setup:**
1. Create artboards for each required size (6.7", 6.5", 12.9", macOS)
2. Add device frames (download from Apple Design Resources)
3. Import screenshots into frames
4. Add headline + subtext overlays
5. Add annotations (arrows, highlights)

**Export Settings:**
- Format: PNG
- Color Space: sRGB
- Resolution: @2x (Retina)
- Compression: Low (preserve quality)

### Step 4: Validate

**Quality Check:**
- Text legible at actual size (open on iPhone/iPad to test)
- No pixelation or blurry UI elements
- Consistent branding across all screenshots
- Annotations don't obscure important UI

**Technical Check:**
- Exact pixel dimensions (use `sips -g pixelWidth -g pixelHeight screenshot.png`)
- File size <5MB per screenshot
- PNG format (no JPEG artifacts)

### Step 5: Upload to App Store Connect

1. Log in to App Store Connect
2. Navigate to app → App Store tab → Screenshots
3. Drag screenshots to appropriate device size slots
4. Reorder to match storytelling sequence
5. Preview on device to ensure readability

## A/B Testing Strategy

### Test Variations (Post-Launch)

**Test 1: Headline Phrasing**
- Variant A: "1,000 Photos Rated in 30 Seconds"
- Variant B: "Cull a Wedding Shoot in Under a Minute"
- Metric: Conversion rate (store views → downloads)

**Test 2: Screenshot Order**
- Variant A: Hero → Before/After → Lightroom → Pricing
- Variant B: Hero → Lightroom → Pricing → Before/After
- Metric: Scroll depth (how many screenshots users view)

**Test 3: Visual Style**
- Variant A: Device frames with gradient backgrounds
- Variant B: Frameless screenshots with text overlays
- Metric: Click-through rate from search results

### Measurement Tools
- App Store Connect → App Analytics → Product Page Optimization
- Run tests for minimum 2 weeks (14 days)
- Minimum 1,000 impressions per variant for statistical significance

## Localization

### Translation Approach

**Headline/Subtext:**
- Translate all overlay text to target languages
- Maintain same character length (adjust font size if needed)

**Screenshots:**
- Option 1: Translate UI text in screenshots (requires rebuilding screens)
- Option 2: Keep English UI, translate overlay text only (faster)
- Recommendation: Option 2 for v1.0, Option 1 for major markets (ES, FR, DE, JP)

### Language-Specific Considerations

**Spanish:**
- Headline may be longer (e.g., "1,000 fotos" vs "1,000 photos")
- Adjust font size to 90% if text wraps

**German:**
- Long compound words (e.g., "Bildauswahlsoftware")
- Use shorter headline variants (e.g., "Schnelle Fotoauswahl mit KI")

**Japanese:**
- Vertical text may be preferred for some layouts
- Ensure font supports Kanji, Hiragana, Katakana

## Common Mistakes to Avoid

### 1. Text Too Small
- Test on actual device, not just desktop monitor
- Minimum font size: 24pt for subtext, 48pt for headlines

### 2. Too Much Information
- Don't cram multiple features into one screenshot
- One key message per screenshot

### 3. Inconsistent Branding
- Use same color palette across all screenshots
- Maintain consistent device frame style

### 4. Low-Contrast Text
- Avoid white text on light backgrounds
- Add semi-transparent overlay behind text if needed

### 5. Outdated UI
- Screenshots must match current app version
- Update screenshots with each major redesign

### 6. Wrong Aspect Ratio
- Don't stretch or crop screenshots to fit
- Design for each size natively
