# Kull App Icon Specifications

## Overview

Kull requires app icons for both macOS and iOS platforms. All icons must be provided in PNG format with no transparency (unless specifically noted).

## Icon Design Guidelines

### Visual Identity
- **Primary Element:** The "K" lettermark from Kull branding
- **Color Scheme:**
  - Primary: Deep blue (#0066CC)
  - Accent: Electric cyan (#00D4FF)
  - Background: Gradient or solid color
- **Style:** Modern, clean, professional
- **Metaphor:** Photography, AI intelligence, speed/efficiency

### Design Principles
1. **Simplicity:** Icon should be recognizable at 16x16 pixels
2. **No Text:** Avoid small text (illegible at small sizes)
3. **Contrast:** Ensure icon stands out on both light and dark backgrounds
4. **Uniqueness:** Differentiate from Adobe Lightroom, Capture One, Photo Mechanic

### Platform-Specific Considerations

**macOS:**
- Slightly more detailed (users see larger icon in Dock)
- Support for light/dark mode variants
- Rounded square with subtle shadow/depth

**iOS:**
- Simpler, bolder design (visible on small screens)
- Fully rounded corners (iOS applies mask automatically)
- High contrast for Home Screen visibility

## Design Tools & Export

### Recommended Tools
- **Sketch:** Use slices for batch export
- **Figma:** Export all sizes at once with export settings
- **Adobe Illustrator:** Use "Export for Screens" feature
- **Photoshop:** Use "Generate Image Assets" workflow

### Export Settings
- **Format:** PNG
- **Color Profile:** sRGB (not Display P3 or Adobe RGB)
- **Bit Depth:** 24-bit (RGB) or 32-bit (RGBA for macOS)
- **Compression:** Standard (not progressive)
- **DPI:** 72 or higher (for App Store icon: 144 DPI recommended)

### Automation Script

sips -z 64 64 icon_1024x1024.png --out icon_32x32@2x.png

## Validation Checklist

Before submitting to App Store, validate icons:

1. **Size Validation:**
   - Run: `sips -g pixelWidth -g pixelHeight icon_*.png`
   - Ensure each icon matches expected resolution

2. **Alpha Channel Check:**
   - iOS icons MUST NOT have transparency
   - Run: `sips -g hasAlpha icon_1024x1024.png`
   - Output should be `hasAlpha: no`

3. **Color Profile Check:**
   - Run: `sips -g profile icon_1024x1024.png`
   - Should be sRGB or Display P3

4. **Visual Inspection:**
   - Open all icons in Preview
   - Check for pixelation, artifacts, incorrect colors
   - Compare small sizes (16x16, 20x20) for legibility

5. **Xcode Validation:**
   - Build in Xcode → Product → Archive
   - Xcode will warn of missing or incorrectly sized icons

## Common Errors & Fixes

### Error: "App icon has alpha channel"
**Fix:** Remove transparency from 1024x1024 icon
```bash
sips -s format png --setProperty formatOptions 70 icon_1024x1024.png --out icon_1024x1024_no_alpha.png
```

### Error: "Icon size mismatch"
**Fix:** Re-export icon at exact pixel dimensions (not 1023x1023)

### Error: "Unsupported color profile"
**Fix:** Convert to sRGB
```bash
sips -m "/System/Library/ColorSync/Profiles/sRGB Profile.icc" icon_1024x1024.png
```

### Error: "Missing icon sizes"
**Fix:** Generate all required sizes using script above

## Testing Icons

### macOS Testing
1. Build app in Xcode
2. Check Finder list view (16x16)
3. Check Dock (128x128)
4. Check App Switcher (Cmd+Tab)

### iOS Testing
1. Install on physical device or simulator
2. Check Home Screen (60x60@3x on iPhone, 76x76@2x on iPad)
3. Check Settings app (29x29@3x)
4. Check Spotlight search (40x40@3x)
5. Check notifications (20x20@3x)

### App Store Connect Testing
1. Upload build via Xcode Organizer or Transporter
2. Check App Store Connect → TestFlight → Build Details
3. Verify 1024x1024 icon appears correctly
